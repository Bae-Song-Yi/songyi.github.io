Description
사내 구내 식당 점심 구독 서비스 개발
Project Background
  • ① PC 환경에 최적화되어 있으며, 모바일 대응이 필요함

    - 식사 대장이 준비되지 않아 식사 사인을 하지 못할 경우 현장에서 내가 주문한 내용을 확인하기 어려움

  • ② 다른 직원들이 메뉴를 착각하여 수령하는 경우가 발생하므로, 신청한 메뉴를 확인할 수 있는 리스트 필요함
  • ③ 복잡한 양도, 인계 프로세스
로그인 화면
  • ① 로그인에 필요한 요소 form태그 안에 입력하여 서버로 전송되어 처리 할 수 있도록 구현했습니다.
  • ② legend 태그 안에 fieldset 태그를 사용해 제목 역할을 부여하였으며, 폼 요소를 그룹화하여 스크립트에서의 접근성을 향상시키고, 사용자 접근성도 높일 수 있도록 구현했습니다.
Calendar 직접 개발
  • ① 부트스트랩을 사용하면 기본 스타일이나 스크립트가 특정 요구 사항과 충돌하여 캘린더의 사용자 정의 기능이나 인터랙션을 제대로 구현하기 어려웠습니다. 이에 보다 유연하게 요구 사항에 맞는 캘린더를 개발하기 위해 부트스트랩을 사용하지 않고 직접 개발하는 방식을 선택했습니다.
